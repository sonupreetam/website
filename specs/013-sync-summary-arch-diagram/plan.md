## Summary

The getting-started page lacked a visual entry point for new users. This feature adds a theme-aware architecture diagram (CSS-only dark/light switching via a reusable Hugo shortcode) and replaces the outdated complyctl quick-start guide with accurate end-to-end instructions for the current release, including cosign signature verification and all four scan output formats.

## Phase 1: Design & Contracts

### Data Model

No persistent data model. The two key entities are purely structural:

**Theme Image Component**

| Attribute | Type | Description |
|-----------|------|-------------|
| `light` | string (Hugo param) | Path or URL to the light-theme image |
| `dark` | string (Hugo param) | Path or URL to the dark-theme image |
| `alt` | string (Hugo param) | Alt text; defaults to `"Architecture Diagram"` if omitted |

Rendered output: two `<img>` elements with CSS classes `theme-image-light` and `theme-image-dark`. Display controlled by SCSS rules:

```scss
// Light mode (default): show light, hide dark
.theme-image-dark  { display: none; }

// Dark mode via explicit Bootstrap attribute
[data-bs-theme="dark"] .theme-image-light { display: none; }
[data-bs-theme="dark"] .theme-image-dark  { display: block; }

// Dark mode via OS preference (no explicit override)
@media (prefers-color-scheme: dark) {
  :not([data-bs-theme="light"]) .theme-image-light { display: none; }
  :not([data-bs-theme="light"]) .theme-image-dark  { display: block; }
}
```

**Architecture Diagram Images**

| File | Purpose |
|------|---------|
| `static/images/complytime-architecture.png` | Light-background variant (~1.2 MB) |
| `static/images/complytime-architecture-dark.png` | Dark-background variant (~1.3 MB) |

Committed as static assets. Not gitignored. Not generated by the sync tool. Maintained manually.

### Contracts

The `theme-image` shortcode is the only interface contract exposed to content authors.

**Shortcode interface:**

```
{{< theme-image light="<path>" dark="<path>" alt="<string>" >}}
```

| Parameter | Required | Default | Description |
|-----------|----------|---------|-------------|
| `light` | Yes | — | Light-theme image path (relative to `static/` or absolute URL) |
| `dark` | Yes | — | Dark-theme image path (relative to `static/` or absolute URL) |
| `alt` | No | `"Architecture Diagram"` | Image alt text for accessibility |

**Output contract:**

```html
<img src="<light>" class="theme-image-light" alt="<alt>" />
<img src="<dark>" class="theme-image-dark" alt="<alt>" />
```

No JavaScript. No Hugo partials called. No external services. Hugo build failure if shortcode file is absent (committed layout override — intentional hard dependency).
