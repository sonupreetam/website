# Feature Specification: Spec-Kit Artifacts as Content

**Feature Branch**: `009-spec-content-type`
**Created**: 2026-03-06
**Status**: Draft
**Phase**: 4 (Content Types)

## Comparative Analysis

### Current State (`complytime/website`)

- No `/specs/` content section
- No mechanism to display `.specify/` artifacts from upstream repos
- Visitors must navigate to raw GitHub files to read specs, plans, or constitutions from complytime repos
- No content type or layout for spec-driven development artifacts

### Target State (`test-website`)

- `content/specs/` directory — Contains fetched `.specify/` artifacts from upstream repos
- Structure: `content/specs/[repo]/constitution.md`, `spec.md`, `plan.md`
- Content generated by the Go sync tool (Feature 6), which checks each repo for a `.specify/` directory
- Browsable at `/specs/[repo]/` on the website
- Makes specifications visible and discoverable without requiring visitors to navigate raw GitHub

### Delta

| Item | Action | Details |
|------|--------|---------|
| `content/specs/` | Add | New content section for spec-kit artifacts |
| `content/specs/_index.md` | Add | Section index page listing all repos with specs |
| `content/specs/[repo]/*.md` | Generated | Constitution, spec, plan from upstream repos' `.specify/` directories |

### Conflicts

- None for the content directory — it is net-new.
- The URL path `/specs/` must not conflict with any existing routes. Current routes are `/`, `/docs/`, `/privacy/`. No conflict.
- The generated Markdown files need valid Hugo frontmatter. The sync tool (Feature 6) must add appropriate `title`, `date`, and `weight` frontmatter to each file.

## Acceptance Criteria

1. `content/specs/_index.md` exists with appropriate section frontmatter
2. After running the sync tool, `content/specs/[repo]/` directories are created for repos that have `.specify/`
3. Spec artifacts are browsable at `/specs/[repo]/` with proper rendering
4. Navigation includes a way to reach the specs section (menu or link)
5. Markdown content renders correctly (headings, tables, code blocks, lists)
6. `hugo build` completes without errors with the specs content

## Migration Steps

1. Create `content/specs/_index.md` with section frontmatter
2. Verify the sync tool (Feature 6) generates spec content correctly
3. Test rendering of fetched spec artifacts
4. Optionally add "Specs" to the navigation menu

## Rollback Plan

Delete `content/specs/` directory. No other files are modified.

## Dependencies

- Feature 6 (Go Sync Tool) for automated fetching of `.specify/` artifacts from upstream repos
